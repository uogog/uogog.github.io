NET 链接到标题

docker本身提供了三种默认的网络模式：

NETWORK ID     NAME      DRIVER    SCOPE
5cafb44e403b   bridge    bridge    local
86f5bc4ba7e1   host      host      local
688cd4970295   none      null      local

bridge 就是创建了一个虚拟的网桥，容器通过这个网桥进行通信，默认情况下，容器之间是可以相互通信的，除非你使用了–network=none来禁用网络。 默认的birge不能直接用容器名字进行通信，需要使用容器的IP地址进行通信，如果想要使用容器名字进行通信，可以创建一个用户自定义的网络，使用–network参数指定网络名称即可。

还有什么overlay网络，主要用于Swarm模式下的多主机网络，可以让不同主机上的容器通过一个虚拟网络进行通信。这个到底是什么我还没有深入了解，后续有机会再学习一下。
